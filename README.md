This project rough outline of the classes involved in a larger application.

You can see from the stack trace in the sample app that the stacktrace is listing a line that does not exist in IndentSpanWatcher. That line of code is actually in BeginningFormatEnabledWatcher

`at com.example.textspanwatcher.IndentSpanWatcher.<init>(SourceFile:37)`

```kotlin
class BeginningFormatEnabledWatcher: TextSpanWatcher() {
  init {
    // Crashes with com.example.textspanwatcher.IndentSpanWatcher@2e602c6
    check(!this.toString().contains("IndentSpanWatcher")) { this.toString() }
  }
```

It seems like R8 causes these classes to be mixed up. You can see in the generated bytecode that two instances of IndentSpanWatcher are created, instead of one IndentSpanWatcher and one BeginningFormatEnabledWatcher:

```kotlin
override fun setListeners() {
    if (quoteAllTheThings.m1) {
      text.addTextSpanWatcher(IndentSpanWatcher())
      text.addTextSpanWatcher(BeginningFormatEnabledWatcher())
    }
  }
```
